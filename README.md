# @shapeshift-labs/frontier-canvas

Renderer-neutral infinite canvas primitives for Frontier apps. The package owns camera math, viewport normalization, grid materialization, snapping, hit testing, tool dispatch, patch plans, and frame records without importing DOM, React, canvas, WebGL, scene, virtual, state, or agent packages.

- npm: [`@shapeshift-labs/frontier-canvas`](https://www.npmjs.com/package/@shapeshift-labs/frontier-canvas)
- source: [`siliconjungle/-shapeshift-labs-frontier-canvas`](https://github.com/siliconjungle/-shapeshift-labs-frontier-canvas)

## Usage

```ts
import {
  createMarqueeSelectCanvasTool,
  createCanvasSurface,
  createCanvasToolRegistry,
  createPanCanvasTool,
  createSelectCanvasTool,
  createZoomCanvasTool,
  materializeCanvasFrame,
  planCanvasFitBounds,
  planCanvasZoomAt
} from '@shapeshift-labs/frontier-canvas';

const surface = createCanvasSurface({
  id: 'editor.canvas',
  statePath: '/editor/canvas',
  document: {
    layers: [{ id: 'shapes', order: 1 }],
    items: [{ id: 'node:1', x: 10, y: 20, width: 120, height: 80, layer: 'shapes' }]
  },
  session: {
    camera: { x: 0, y: 0, zoom: 1 },
    viewport: { width: 1280, height: 720 },
    grid: { enabled: true, size: 16, subdivisions: 4, snap: true },
    activeToolId: 'canvas.select'
  },
  tools: [createSelectCanvasTool(), createMarqueeSelectCanvasTool(), createPanCanvasTool(), createZoomCanvasTool()]
});

const frame = materializeCanvasFrame({ surface });
const zoom = planCanvasZoomAt({
  camera: surface.session.camera,
  viewport: surface.session.viewport,
  screen: { x: 640, y: 360 },
  factor: 1.2
});
const fit = planCanvasFitBounds({
  bounds: { x: 10, y: 20, width: 120, height: 80 },
  viewport: surface.session.viewport,
  camera: surface.session.camera,
  padding: 24
});

const registry = createCanvasToolRegistry({ tools: surface.tools });
const record = registry.dispatch({
  event: { type: 'pointerdown', screen: { x: 120, y: 120 } },
  context: {
    surface,
    session: surface.session,
    camera: surface.session.camera,
    viewport: surface.session.viewport,
    frame
  }
});
```

The dispatch record contains patch-shaped output instead of host mutations. DOM, SVG, Canvas2D, WebGL, or an agent tool adapter can consume the same frame and records.

## Research Transfer

The package keeps the proven infinite-canvas ideas as serializable data rather than renderer-owned state:

- tldraw-style page/world coordinates, viewport coordinates, and screen coordinates map to `screenToWorld`, `worldToScreen`, and `canvasViewportWorldRect`.
- D3/Fabric/Konva-style scale+translate math maps to immutable camera plans and pointer-anchored `planCanvasZoomAt`.
- React Flow and tldraw fit/constraint ideas map to `planCanvasFitBounds` and `constrainCanvasCamera`.
- Pixi viewport's removable behavior plugins map to structural tools such as `createPanCanvasTool`, `createZoomCanvasTool`, and `createMarqueeSelectCanvasTool`.
- Excalidraw-style app state, selection state, grid state, and tool state stay explicit in `FrontierCanvasSession`, frame records, and patch outputs.

## Tool Extensibility

Tools are structural records with optional handlers. A future `frontier-tools` bridge can expose the same tool definitions to agents without making this package depend on it.

```ts
registry.register({
  id: 'canvas.measure',
  events: ['pointerdown'],
  reads: ['/editor/canvas/document/items'],
  writes: ['/editor/canvas/session/measureStart'],
  requires: ['canvas.measure'],
  handlers: {
    pointerdown(ctx, event) {
      return {
        patches: [{
          op: 'set',
          path: '/editor/canvas/session/measureStart',
          value: event.world
        }]
      };
    }
  }
});
```

Built-in helpers cover panning, wheel zoom, click selection, and marquee selection, but the registry does not special-case tool ids. Applications can add brush, connector, resize, lasso, text, minimap, whiteboard, or game-editor tools later.

## API Surface

- `screenToWorld`, `worldToScreen`, and `canvasViewportWorldRect` define the world/screen transform.
- `planCanvasPan`, `planCanvasZoomAt`, `planCanvasWheelZoom`, and `planCanvasFitBounds` return serializable patch plans for camera updates.
- `constrainCanvasCamera` applies zoom limits and bounded camera movement without mutating host state.
- `materializeCanvasGrid` returns visible major/minor grid lines for the current viewport.
- `materializeCanvasFrame` returns visible items, screen bounds, selected handles, grid lines, and summary data.
- `hitTestCanvas` resolves topmost items from world or screen coordinates.
- `selectCanvasItemsInRect` supports box/lasso-style selection over frame or item data.
- `createCanvasToolRegistry`, `defineCanvasTool`, `createPanCanvasTool`, `createZoomCanvasTool`, `createSelectCanvasTool`, and `createMarqueeSelectCanvasTool` provide extensible tool dispatch.
- `createCanvasStateLayout`, `createCanvasStateLayoutForSurface`, `classifyCanvasStatePath`, and `partitionCanvasPatches` split document/session patches into CRDT-syncable and local-only buckets without importing CRDT packages.
- `createCanvasRegistryGraph`, `traceCanvasImpact`, `createCanvasProof`, and JSONL helpers make canvas surfaces inspectable by Frontier tooling.

## CRDT And Local State

Canvas state is deliberately path-partitioned. Document paths such as `/canvas/document/items` and `/canvas/document/layers` default to `crdt`; session paths such as `/canvas/session/camera`, `/canvas/session/selectedIds`, `/canvas/session/hoverId`, and `/canvas/session/pointer` default to `local`.

Apps can promote or demote either side with a state layout:

```ts
const layout = createCanvasStateLayoutForSurface(surface, {
  scopes: {
    session: 'crdt',      // shared camera / selection session
    pointer: 'local'      // keep pointer capture local even in a shared session
  }
});

const { crdt, local } = partitionCanvasPatches(toolRecord.patches, layout);
```

The CRDT adapter should consume the `crdt` patches. The local store should consume the `local` patches. Tool registries and handlers remain runtime/static registry data; sync the JSON document/session state and action records, not handler functions.

## Boundary

`frontier-canvas` depends only on `@shapeshift-labs/frontier`. Integrations with `frontier-virtual`, `frontier-scene`, `frontier-view`, `frontier-dom`, `frontier-tools`, `frontier-policy`, `frontier-history`, and `frontier-test` should stay structural or live in higher packages/adapters.
